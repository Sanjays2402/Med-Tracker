## Notable design decisions

### `dose-export-csv`
- First @med/utils data-export module with a deliberate ROUND-TRIP
  parser (`parseDoseCsvExport`).
- Three layouts: MED_TRACKER (full, round-trippable), WALGREENS,
  CVS. Each chain layout matches the chain's published columns,
  not what FHIR thinks the columns should be.
- Pharmacy-status mapping (TAKEN / TAKEN-LATE / MISSED / SKIPPED /
  PENDING) is the documented retail label, not the DoseStatus enum.
- RFC-4180 quoting; empty cells emit as bare `,,` rather than `""`
  because pharmacy parsers reject quoted empty strings.
- BOM + CRLF + LF tolerance in the parser because patients hand-
  edit CSV in Excel-on-Windows.

### `refusal-reason-suggest`
- First rule-based suggestion pipeline in @med/utils.
- 5 rules in strict priority order: NPO (0.95) > pause (0.9) >
  out-of-supply (0.85) > sleeping (0.7) > pattern (0.4-0.65 scaled).
- Sleeping rule uses `quiet-hours.isInQuietHours` so wrap-midnight
  semantics match the reminder engine.
- Pattern tie-break: tolerability reasons (nausea > side-effect)
  beat declined so the more-actionable de-prescribing signal
  surfaces first.

### `followup-digest-html`
- First HTML email-body fragment renderer.
- Mirrors followup-overdue-digest's null short-circuit + subject +
  most-overdue-leads opener (verified by parity test).
- All styles inline (Gmail strips `<style>`); colour palette
  matches Tailwind for dashboard consistency.
- HTML-escape via standard 5-char replace; verified by an explicit
  `<script>`-tag patient-name test.

### `refusal-trend-summary-html`
- First module to produce BOTH a chart-component-ready data
  payload AND a rendered HTML fragment from one call.
- Chart payload is shape-compatible with Recharts/Apex/etc with
  no further mapping.
- Inline HTML sparkline (24px-tall scaled bars) so email previews
  get a visual signal when canvas charts can't render.
- ASCII bar fallback (U+2581..U+2588) ships alongside.
- TOLERABILITY LEAD chip overrides direction chip when
  `risingTolerability` fires — actionable signal first.
- `actionableOnly` (default true) hides non-actionable rows from
  the email body BUT keeps them in the sparkline payload.

### `regimen-snapshot-archive-restore`
- SECURITY-FIRST ordering: verify BEFORE diff. All 5 failure modes
  (malformed / bad-version / signature-mismatch / payload-tampered
  / secret-too-short) pass through verbatim.
- 8 RestoreItemAction values cover the full diff space.
- Single-field divergence collapses to a focused action; multi-
  field divergence escalates to `collision` with field-name list
  for caller adjudication.
- `currentOnly` surfaces medications added after the snapshot;
  hasChanges does NOT flip to true based on currentOnly alone.
- Schedule comparison uses identical normalisation to regimen-
  snapshot-archive (times sorted, daysOfWeek sorted, intervalHours
  / cronExpression null-normalised).
- Plan is a PROPOSAL — module never mutates the regimen or writes
  to a DB.
