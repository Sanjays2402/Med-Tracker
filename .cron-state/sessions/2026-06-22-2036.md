## Notes per feature

### #196 `regimen-snapshot-archive-history-rollup-csv-export-merge-anonymise-key-rotate-html`

FIRST HTML render of the rotation mapping in the package. Use case: clinic auditor reviewing the rotation by eye needs a table view, not a JS struct.

- Defaults to **NON-PHI output** (no original patient ids / names) — fragment is filable outside the patient chart per HIPAA safe harbour (45 CFR 164.514).
- `includeOriginalIds=true` opt-in adds source columns for the PHI variant.
- Banner with count + NO-OP ROTATION + COLLISION DETECTED / NO COLLISIONS chips — auditor sees verdict before scrolling.
- Per-row no-op "unchanged" chip distinguishes pseudonym-didn't-change rows from rotated rows.
- Three sort orders (old-pseudonym default, new-pseudonym, patient-id which throws when PHI not enabled).
- `renderRegimenHistoryAnonymiseKeyRotateHtmlChangesOnly` is the canonical "audit-binder ready" preset (no-op rows hidden, non-PHI, lex-sorted, fragment).

### #197 `dose-export-csv-import-roundtrip-validator-summary-text-slack-thread-batcher-quiet-hours-coverage-report`

FIRST JSON coverage report for the quiet-hours decision stream.

- Walks (runAt, decision) pairs; rolls posted / deferred / suppressed counts + per-reason post-now breakdown.
- `deferralLatenciesMs` (min/max/mean) reports actual deferral durations. Negative latencies (broken upstream config) drop out of the math but the decision still counts in the deferral total.
- Three misconfiguration flags: `channelIsAlwaysDeferring`, `channelIsAlwaysSuppressing`, `channelIsAlwaysPostingNow`.
- `detectQuietHoursMisconfiguration` surfaces one condition as a string for the dashboard. **Multi-window-labels check fires first** — usually the root cause of always-deferring / always-suppressing patterns.
- `summarizeQuietHoursCoverageReport` one-liner for the cron log.

### #198 `followup-digest-text-html-bundle-i18n-multi-locale-cron-batcher-html-mailer-bcc-coverage-report`

FIRST JSON coverage report for the BCC envelope stream.

- Translates the basic module's `Map<string, number>` fanOutByAddress to a JSON-safe `{address, count}[]` sorted DESC by count, ASC by address for ties. (Verified JSON roundtrip in tests.)
- `declaredDestinations` input is ground truth for `unusedBccAddresses` — without it the unused list is empty.
- `topNFanoutAddresses` for the "loudest addresses" widget.
- `detectFollowupDigestBccMisconfiguration` surfaces three conditions: zero-headers-with-declarations (scope filters too narrow), unused addresses (per-caregiver scope likely filtered them out), extreme fan-out skew (>75% on one address with 3+ distinct addresses).

### #199 `refusal-reason-suggest-i18n-rollup-html-print-cover-sheet-binder-spine-i18n`

FIRST chrome-string i18n bundle in the package (the existing `refusal-reason-suggest-i18n` bundles cover EXPLANATION strings, not chrome).

- Bundle pattern mirrors the existing i18n layer: `{ locale, strings: Partial<...> }` with per-key English fallback so contributor-submitted incomplete locales never blank the spine.
- **Five built-in bundles** ship out of the box: en-US, es-419 (covers es-ES via region-strip), fr-FR (covers fr-CA), de-DE, hi-IN (devanagari).
- `renderLocalisedRefusalReasonSpine` wraps the base renderer: drops the base's hard-coded `includePanelSize` emission (English-only), then re-injects the localised `<count> <unit>` label with the bundle's strings before the closing `</div></section>`. Uses the base spine's font metrics (cross-axis font sizing heuristic) so the injected label is visually identical to what the base would have emitted in English.
- `pickBuiltInSpineBundle` + region-strip lookup: 'es-MX' → 'es' → es-419; unknown locale → en-US.
- `validateSpineI18nBundle` returns missing keys for CI checks of contributor submissions.

### #200 `prescriber-contact-card-emergency-card-pdf-two-up-watermark-roster-toc-print-only`

FIRST standalone-TOC variant in the package (existing TOC modules always emit combined document or its HTML companion).

- Use case: clinician auditing the binder index doesn't want to re-print every card; clinician photocopying a roster lookup for a colleague wants ONLY the index.
- Header strip rewritten "Page 1 of N+1" → "Page 1 of 1" for standalone-document consistency.
- Footer block rewritten "Document N pages total" → "Index only (binder spans N pages)" so the clinician knows they're holding the index, not a partial copy.
- **Per-entry pageNumber values STAY pointing into the underlying binder** (Page 3, Page 5, etc.) — the index is still useful as a binder-lookup reference. No rewrite to 1.
- Defensive footer synthesis if upstream TOC ever stops emitting a footer block.
- `combinedDocumentPageCount` preserved on the result so UI banners can say "you are looking at the index for an 8-page binder".
