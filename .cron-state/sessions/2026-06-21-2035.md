## Module notes

### `dose-export-csv-import-roundtrip-validator`

First diff-aware re-import path. Risk classifier collapses every diff
to ONE of four tiers (note-only / status-edit / structural / mixed)
so the adjudication UI maps to chip colors without re-parsing field
lists. Note-only is the auto-acceptable safe tier — the UI's
"auto-accept low-risk changes" toggle bulk-applies them.
`applyAcceptedDiffs` returns a NEW array (no source mutation) because
the adjudication UI runs in React + Redux flows where source
mutation breaks every memoisation. Empty-note vs missing-note treated
identically so a clean round-trip produces zero spurious diffs.

### `refusal-reason-suggest-i18n`

First i18n layer in `@med/utils`. Stable `source` discriminator is the
i18n key, NOT the English text — locale tables are pure JS objects
mapping `source -> ICU template`. Suggester logic stays English-only;
only the picker tooltip is localised. Placeholders extracted by regex
on the suggester's English explanations so we don't have to change
the suggester's public shape. Triple fallback chain:
missing-locale-key → suggestion's own English; unrecognised-English-
template → same; unknown-placeholder → leave as `{name}` + record in
`missingPlaceholders`. Ships a `validateRefusalReasonI18nTable`
helper for CI-grade locale file checking.

### `followup-digest-text-html-bundle`

First composition that GUARANTEES drift-free output across two
single-body builders. Naive composition has two drift modes
(section limits diverging, independent null short-circuits). This
wrapper threads shared options to BOTH builders and uses ONE null
check. Returned shape ships `text` / `html` top-level + an explicit
`alternatives` array for the SMTP providers (SES raw, mailgun raw)
that need the MIME shape directly. Defensive throw when text-digest
non-null but html-digest null — catches future divergence of the two
predicates instead of silently shipping a half-built bundle.

### `prescriber-contact-card-emergency-card`

First layout variant of an existing card type rather than a wrapper.
ED handoff context inverts the standard hierarchy: ON-CALL number
leads at 32px, daytime drops to a fallback line, fax / NPI / address
/ scheduling URL drop entirely. Specialty must be present —
without it the ED clerk cannot route the patient; missing-specialty
drives a warning. Text render uses centred ASCII with `===`
separators so the hero phone line stays visually dominant even on a
black-and-white pharmacy printer. HTML render adds a red border + red
EMERGENCY CONTACT header. `source` PrescriberContactCard preserved as
back-reference for downstream consumers that need the dropped fields.

### `regimen-snapshot-archive-history-rollup`

Rolls a chronological list of snapshots into a per-medication
add/remove/change timeline. Uses `diffRegimenSnapshots` PAIRWISE for
every adjacent pair so the rollup is byte-for-byte consistent with
single-pair diffs (future diff fix automatically fixes the rollup).
Cycled medication detection (added → removed → added) is a specific
clinical signal worth its own list. Re-add preserves firstSeen as the
ORIGINAL first appearance — cumulative tenure is what clinicians
read. `name` always carries the MOST RECENT name. Prescriber +
pharmacy joins deliberately ignored — they churn for non-clinical
reasons and would crowd out the strength/presence signals the
prescriber cares about.
