## Features shipped

### 1. prescriber-contact-card (3ce449c)

Wallet-printable prescriber contact block. Phone normalisation
includes 10-digit US -> +1 E.164 promotion (separate `.e164` field
from raw `.digits` so saved-to-contacts cards dial correctly
internationally). renderWalletCard produces 8-line max 32-col text
sized for a 3.5x2" business card. renderVcard emits RFC 6350
(vCard 4.0) subset with CRLF line endings + 75-char folding.
Warnings accumulate on the card object (phone invalid, NPI
failed Luhn, no contact methods) rather than throwing.

### 2. appointment-followup-tracker (c1a333d)

Tracks recommended follow-up appointments / labs / imaging /
referrals with due-by-date semantics, completion log, cancellation
log. Per-kind warn windows (lab=7d, referral=21d for scheduling
lead, others=14d). graceDays controls the "Overdue by N days (past
grace window)" message escalation without changing the status
bucket. Completion wins over cancellation when both arrive.
deriveFollowupFromRecommendation translates clinic shorthand
("3 months", "6 weeks") into absolute dueAt; day-of-month clips
to last-of-month when target month is shorter.

### 3. medication-refusal-log (603d16a)

Per-dose refusal log with 10-code vocabulary. REFUSAL_EXCLUDED_
REASONS = {npo, prescriber-paused, out-of-supply} — these get
excluded from the honest adherence denominator. sleeping
deliberately NOT excluded (it's a real adherence problem signaling
a schedule change). REFUSAL_TOLERABILITY_REASONS = {nausea,
side-effect} flags de-prescribing candidates when cluster on a
medication. computeAdherenceWithRefusals returns BOTH strict
(PBM-facing) and honest (patient + prescriber facing). When honest
denominator collapses (every dose was NPO), reports 1 — no alert
on "no opportunity to fail" windows. Validation never throws;
per-row error codes let UI surface bad rows without losing the
batch.

### 4. regimen-load-trend (0057b41)

Rolling-window trend of RegimenLoadScore snapshots. Mirrors
pdc-trend's structure for shared dashboard chart components.
CRITICAL INVERSION: positive delta = burden RISING = bad
(pdc-trend's positive delta = improving = good). Per-component
direction surfaced so the "what drove the climb?" panel can say
"dosing rising, pills stable". Distinct-days gate prevents a single
snapshot replicated across overlapping windows from being mistaken
for a trend. Each window's number is the MEAN (robust to single-
day cost spikes like deductible resets), not the latest snapshot.
Does NOT recompute scores — snapshots are authoritative because
historical pill-burden / cost state isn't preserved at runtime.

### 5. dose-batch-export (c7a414c)

First interop-export module. Generates FHIR R4 Bundle of
MedicationAdministration resources from internal Dose +
Medication. Bundle type 'collection' (read-only export, NOT
'transaction' — consumer re-wraps on their side if ingestion
desired). Status mapping: taken/late -> completed; skipped ->
not-done + statusReason 'patient-skipped'; missed -> not-done +
statusReason 'missed'; scheduled -> in-progress (dropped by
default — exports normally ship realised history). Route mapping
via HL7 v3 RouteOfAdministration codes (PO/IM/TD/IH/OPHTH/PR/SC).
Entries sorted by effectiveDateTime asc for stable diffable output.
fullUrlBase option for canonical URIs on a known FHIR server.
