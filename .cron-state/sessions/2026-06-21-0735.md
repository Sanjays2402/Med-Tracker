## Gotchas learned

- `as const` on a DEFAULT_WEIGHTS object narrows fields to literal
  values and makes a Partial<T> merge return type unassignable. Use
  an explicit interface for "defaults + caller overrides" patterns.
- `expireMinutes <= delayMinutes` is a structural validation error,
  which means the "tier expired before firing" state can never appear
  in a static simulation built from a valid policy. Removed the dead
  `expired: boolean` from SimulationTierEvent before commit.
- `ValidationResult` / `ValidationError` are dangerously generic
  names. food-windows already exported both; my new module collided
  on first build. Fifth `<Module>Foo` rename in the autoship history
  — pattern is now "prefix the NEW module's type with its domain
  noun at first-write".
- `require()` in vitest TS test files needs `@types/node`, which the
  repo does not have. Killed the dead `require()` line in the
  regimen-snapshot test; vitest auto-imports from ../src work fine.
