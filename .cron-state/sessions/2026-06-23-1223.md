## Notes per module

### `regimen-snapshot-archive-history-rollup-csv-export-merge-anonymise-key-rotate-bulk-cli-summary` (#221)

First multi-line CLI summary in the package. Emits N transition lines
plus one batch verdict line. Transition tags carry the epoch labels
(`[key-rotate epoch=secret-2022->secret-2023]`) so a stack of nightly
log files can be grep-ed by epoch.

Batch line shape: `[key-rotate-bulk] epochs=N transitions=N patients=N
noop_transitions=N collisions_total=N verdict=V`

Verdict precedence (worst wins): widen-hash > empty-cohort > ship-safe
> no-op. `suppressNoOpTransitions` hides noisy unchanged-secret rows
while keeping structured summaries intact.

23 tests.

### `dose-export-csv-import-roundtrip-validator-summary-text-slack-thread-batcher-quiet-hours-calendar-html-printable` (#222)

First print-only HTML variant with @page CSS preset. Composes the
dashboard calendar so per-cell datum stays consistent, then post-
processes for paper:
- monochrome palette (white background across every rule)
- current-day star stripped via deterministic transform
- overlay CSS suppresses the `--current` outline with `!important`
- non-default rule labels bolded by default

@page presets: 8.5in × 11in (us-letter) or 210mm × 297mm (a4).
`printedAt` emits "Printed YYYY-MM-DD" line using Intl.DateTimeFormat
in the right timezone. `wrapHtmlDocument` defaults TRUE (printable
pages are standalone).

`extractQuietHoursCalendarHtmlPrintableLines` emits plain-text per-day
lines for log review without rendering HTML.

30 tests.

### `followup-digest-text-html-bundle-i18n-multi-locale-cron-batcher-html-mailer-bcc-tier-policy-coverage-report-warnings-html` (#223)

First severity-tiered warning chip render. Classifies warnings into:
- `always-critical` RED — channel page-worthy on every envelope
- `always-tier` AMBER — channel is one-dimensional
- `unused-destination` GREY — cleanup, not action

Each chip carries the severity label + human label + (for
unused-destination only) the address split into a monospace span so
the on-call can copy/paste it.

Empty state renders as a single green "All checks passed" chip;
`suppressEmptyState=true` opts out. Unknown warning strings fall into
`unused-destination` as graceful degradation.

`extractBccTierPolicyCoverageUnusedDestinations` returns sorted
addresses for cleanup tooling.

24 tests.

### `refusal-reason-suggest-i18n-rollup-html-print-cover-sheet-binder-spine-batch-csv-manifest-anonymise` (#224)

First PHI-anonymising spine manifest. Hashes patient names BEFORE the
CSV is built so a sticker-paper print shop or label-printing pipeline
(Avery / Brother) gets the geometry it needs without PHI in transit
or at rest.

ASYNC — uses Web Crypto `subtle.sign` with HMAC-SHA-256, same pipeline
the regimen-snapshot anonymise module already uses. Two name
strategies: `hashed` (default, "spine-7a3f1b2c") and `redacted`
(literal "REDACTED").

`hmacSecret` enforced at ≥ 32 chars. Each distinct source name hashed
ONCE. Collision detection flags `collisionDetected` when two source
names map to the same pseudonym.

THREE outputs:
1. `manifestCsv` — third-party-safe
2. `sheetSummaryCsv` — pass-through (no PHI)
3. `nameLookupCsv` — IN-HOUSE only, source-to-pseudonym mapping for
   label-print-error reversal

32 tests.

### `prescriber-contact-card-emergency-card-pdf-two-up-watermark-roster-toc-html-anchored-search-input` (#225)

First search-input scaffolding in the package.

**HONEST scope**: pure CSS cannot read the input's value into a
selector so substring filtering by typed text is NOT shipped. What IS
shipped:
- `<input type="search">` at the top of the TOC wrapper (first
  focusable element)
- per-row `data-toc-name` (lowercased displayName) + `data-toc-specialty`
  (lowercased, "other" when null) for substring matching
- `<datalist>` with one option per prescriber for browser autocomplete
  (zero JS needed)
- `:placeholder-shown` sibling selector covering the empty-state
  baseline (every row visible when input empty)
- `:not(:placeholder-shown)` sibling selector as the hook the host
  page extends with a 5-line `oninput` handler

Graceful degradation: when no host hook is wired, every row stays
visible AND the browser's built-in find (Cmd-F) highlights matches via
the data attributes.

`buildEmergencyCardSearchInputAttributeFragments` returns per-cardIndex
HTML attribute fragments ready to splice into the host's card markup
so cards and TOC rows filter on the same hooks.

## Honest-scope call

#225 (search-input) explicitly DID NOT pretend to ship pure-CSS
substring filtering. The prompt asked for a "CSS-only `:not()` match"
but true substring filtering by arbitrary typed text is impossible in
pure CSS — CSS cannot read input values into selectors. The shipped
module is explicit about this in the docstring and ships the
scaffolding (input + data attributes + datalist + baseline CSS) that a
5-line host hook completes. Better to ship an honest scaffold than a
fake-CSS-only filter that doesn't work.
