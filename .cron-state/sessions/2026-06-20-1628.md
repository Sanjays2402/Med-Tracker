## Design notes preserved for future ticks

### dose-time-drift

- Operates on `DoseHistoryEntry[]` so it plugs directly into the
  import pipeline and the aggregator output.
- Filters: missing takenAt, missing dueAt, skipped — all excluded.
  When called via `computeDoseTimeDrift(medicationId, entries)`, foreign-
  medicationId entries are also excluded.
- Noise clipping (`noiseClipMinutes` default 240) bounds the magnitude
  of a single delta but preserves the sample. Important for keeping
  the median robust on small histories where one wild outlier would
  otherwise swing the verdict.
- Confidence blends sample-size (caps at 2x minSamples) and a MAD-based
  consistency score. Direction reverts to 'insufficient' if the slope
  is non-trivial but the consistency is below `minConsistency`
  (default 0.4) — prevents recommending a 60-min shift when half the
  doses are at -30 and half at +90.
- `recommendedTimeShiftMinutes` is rounded to `shiftStepMinutes`
  (default 5) for a UI-friendly suggestion.

### regimen-change-diff

- Schedule comparison uses a `scheduleKey` normalizer (sorted times,
  sorted daysOfWeek, kind, interval, cron, enabled) so reordered times
  do NOT show up as a change.
- A schedule whose `kind` changes (daily -> interval) produces both a
  schedules-removed and a schedules-added diff entry; the bullet
  renderer prints both so the UI is unambiguous.
- `treatInactiveAsRemoved` (default true) maps active=true->false into
  a removal with `reason='inactive'` so the UI can show "discontinued"
  vs "vanished entirely" (the latter only happens via hard-delete).
- `fields` allow-list lets the dashboard restrict to a subset (e.g.
  only show schedule changes on the calendar page, only show supply
  changes on the refill page).

### caregiver-permission-matrix

- Scope -> capability map is keyed by
  `NonNullable<CaregiverShare['scopes']>[number]`. Adding a new scope
  in @med/types is a type error here until it's handled in the map.
- Deny-wins-over-grant. A medication marked deny=['view-medications']
  is invisible even if the global scope grants view-medications.
- Expired shares produce a matrix with expired=true, empty global,
  empty perMedication. Every helper short-circuits to false / empty;
  no per-med override can grant access on an expired share.
- Capability vocabulary is intentionally explicit — no implicit
  "admin" capability that could grant unexpected access.

### insurance-tier-pick

- Approximation around deductible: charge fullPriceCents while
  remaining deductible >= fullPriceCents on this fill, else copay.
  Flag `deductible-applies` is set whenever remaining > 0 so the UI
  can disclose the assumption ("estimate may differ from actual
  charge once deductible is exhausted").
- Tiebreak on equal 30-day cost: offerings WITHOUT prior-auth or
  step-therapy flags win; then lexicographic name.
- Mail-order + daysSupply >= 90 produces both 'mail-order-discount'
  and 'ninety-day-pack' flags so the UI can compose a "Switch to
  mail order for $X less" nudge with the specific reason.
- `preferMailOrderForLongTerm=false` suppresses the 90-day-pack flag
  but keeps mail-order-discount (callers might want to disable the
  pack nudge for a patient who has trouble managing 90-day supplies).

### caregiver-summary-rollup

- Reuses `DigestInput` from caregiver-digest.ts so a single SQL query
  can be fanned-out into both per-patient emails and a combined
  household summary. No new query patterns.
- Singular vs plural is hand-rolled (no Intl.PluralRules) so the
  messages render the same in every environment.
- `attentionMedications` is sorted by PDC ascending and capped at
  `perPatientMissedLimit` (default 3) with an "and N more" tail.
- Subject text conditionally includes the attention clause only when
  at least one patient has a med below threshold ("avg 92% adherence"
  vs "avg 92% adherence, 2 need attention").
- Average PDC across patients is the mean of per-patient PDCs (not
  weighted by med count); this matches what a caregiver intuitively
  expects when scanning a small household.
