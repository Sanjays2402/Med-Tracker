## Notes per feature

### 1. bulk-cli-summary-json
FIRST structured-JSON variant of a CLI summary in the package. Parses leading-tag from each transition line via a tight regex (`^(\[[^\]]+\])`) and produces typed per-transition + batch entries shaped for direct JSON.stringify. Defensive: malformed lines fall back to default tags rather than crash.

Helpers: `joinAsNdjson` (one JSON object per line: transitions first then batch — analytics pipelines ingest NDJSON streams directly), `filterByVerdict` (dashboard "show me every widen-hash transition" view), `combine` (multi-cohort combiner with worst-wins batch verdict precedence widen-hash > empty-cohort > ship-safe > no-op matching the underlying bulk semantics).

Round-trip safety verified: every field is JSON.stringify clean; integer fields stay numeric after JSON.parse so a time-series DB ingest doesn't accidentally cast to string.

### 2. quiet-hours-calendar-html-printable-i18n
FIRST i18n bundle on a printable HTML render. Composes the base printable renderer so the underlying cell datum stays English-typed (cells[].dayOfWeek / cells[].rule are still the structured keys for downstream consumers) — only the rendered chrome text changes.

Bundle: days (mon..sun), rules (the four canonical labels), printedPrefix, defaultFooterText. All optional; missing keys fall back to English. Caller-supplied `footerText` override always wins verbatim (no i18n at this layer — explicit copy is the caller's own).

Bold-non-default rule labels survive the rewrite — the renderer walks both bare and `<strong>`-wrapped patterns. `detectCoverage(bundle)` standalone CI gate helper for production rollout.

### 3. coverage-report-warnings-html-print
FIRST print-only variant of a coverage warnings panel. Severity differentiation moves from COLOUR to BORDER WIDTH (3px/2px/1px) + BADGE PREFIX (`[CRITICAL]`/`[CAUTION]`/`[INFO]`) so the signal survives both monochrome printing AND colour-vision differences. Chips stack vertically. Address span loses coloured background, gains hairline 1px grey border, uses tabular-nums.

`@page` CSS sized to US Letter / A4. Optional timezone-aware "Printed YYYY-MM-DD" stamp. `extractLines` per-chip plain-text helper for log review.

### 4. spine-batch-csv-manifest-anonymise-key-rotate
FIRST per-feature anonymise-key-rotate companion (parallel to regimen-history). Runs `exportSpineBatchCsvManifestAnonymise` twice (once per secret) and produces FOUR outputs:

- `oldManifestCsv` + `newManifestCsv`: third-party-safe.
- `rotationLookupCsv`: 3-col mapping (original + old + new pseudonym) — IN-HOUSE only (contains PHI).
- `rotationLookupCsvWithoutOriginalNames`: 2-col mapping safe to share with the third-party printer so they can update their lookup table without seeing source PHI.

Both secrets enforced at >= 32 chars. `oldSecret === newSecret` accepted (not an error) but flagged `noOpRotation=true`. `collisionDetected` = OR of both anonymise results. `nameStrategy='redacted'` always produces `noOpRotation=true`.

### 5. search-input-keyboard-nav
FIRST keyboard-navigation helper. Composes the search-input render and exposes `focusableOrder` (flat array in tab order) + `keyMap` (per-element bindings array).

Default bindings: search input ArrowDown -> first row, Home -> first, End -> last; per row ArrowDown -> next (omitted on last row to allow browser tab-out), ArrowUp -> previous OR back to search, Home/End -> first/last, Escape -> search.

**Honest scope**: ships the SCAFFOLDING (focusable order + keyMap); the keydown handler itself is intentionally NOT shipped. A host page wires it in 8 lines via the supplied `exportAsJson` helper. `suppressHomeEndBindings` / `suppressEscapeBinding` opt-outs for hosts with conflicting bindings.
