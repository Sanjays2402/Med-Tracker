## Notable corrections during the tick

1. **dose-instruction-parser frequency ordering.** Initial pattern
   order put `qd` (which matches bare `daily`) first, so `twice daily`
   was matching as `qd` and the bid pattern never fired. Reordered
   longest-first so `bid` / `tid` / `qid` patterns are tried before
   the bare `daily` pattern. Test suite caught it on first run.
2. **dose-instruction-parser reason extraction.** First pass only
   handled `for X` patterns. Real prescriber strings often write
   `prn pain` / `prn nausea` instead. Added a second matcher.
3. **dose-instruction-parser stop-word list.** Without a STOP_WORDS
   list, `for pain qid` captured `pain qid` as the reason. Added an
   explicit stop list including frequency, route, food, and timing
   keywords.
4. **refill-cost-projector cadence anchor.** Naive implementation
   reset cadence to `from` when `firstFillAt` was in the past, which
   would cause systematic over-counting. Walks forward in
   daysSupply-sized steps to land on or after `from` without drift.
