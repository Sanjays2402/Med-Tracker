## Highlights and design notes

1. **weight-trend** — Needed a fallback for the MAD outlier test
   because on a near-constant window (e.g. nine identical 72 kg
   readings + one 200 kg outlier) MAD evaluates to zero and the
   classic `madFactor * MAD` threshold never trips. Fall back to
   `mean(nonZeroDeviations) / 8` when MAD == 0 so a single bad reading
   still gets flagged.

2. **dose-streak-by-med** — Each medication now gets its own current
   and longest streak with the ISO range of the longest run for
   milestone UI ("Apr 1 — May 12, 42 days"). `streaksAtRisk` is the
   key compositional bit: it returns medications where `current = 0`
   but `longest >= minLongest`, perfect for proactive caregiver
   nudges before a lapse becomes a relapse.

3. **pill-burden** — `parseStrength` exported as a reusable helper.
   Polypharmacy thresholds match the published geriatric definitions
   (5+ polypharmacy, 10+ hyperpolypharmacy). Weekly schedules
   fractionalize correctly; interval schedules expand cleanly; PRN
   schedules are intentionally excluded.

4. **pharmacy-distance-pick** — Composes with `pharmacy-hours.ts`.
   `haversineDistanceKm` exported for reuse in `refill-batching` (a
   future tick could weight pickup batches by physical distance).
   Default `includeClosedFallback: true` so an empty open list still
   surfaces *something* with a `nextOpen` time.

5. **dose-adherence-trend** — Composes with `dose-history-aggregator`
   directly. The least-squares fit excludes buckets with zero
   scheduled doses (so they don't pull the line toward zero). r2 floor
   (`minR2: 0.1` default) flips bouncy series to `'insufficient'`
   even with a non-trivial slope, which prevents false improving /
   declining signals on noisy data. **Pattern for future "trend"
   modules: always pair slope with an r2-based confidence gate.**
