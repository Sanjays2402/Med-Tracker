## What I built

### 1. Prometheus exporter for the bulk CLI summary
Strict exposition-format payload, 9 metric families. Cohort patients/epochs/transitions_total/collisions_total/noop_transitions (4 gauges + 1 counter); per-verdict status (gauge — 1 sample per known verdict, 1 for current); per-transition patients/reshuffled/collisions (gauges + counter). Label-value escaping per spec. `extraLabels` for per-cohort tagging on a shared /metrics endpoint with reserved-label clash detection. Metric prefix validated against the Prometheus naming rules. Sort-on-emission so the payload is byte-stable regardless of caller insertion order.

### 2. Multi-page printable quiet-hours calendar
Form-feed (ASCII 0x0C) page separators between per-region documents. Per-region options inherit from `baseOptions` with explicit REPLACE semantics on each field (so a New York region doesn't inherit Berlin's `defaultWindow`). `wrapEachPageAsDocument` defaults TRUE; `pageSeparator` overridable / suppressible. `splitQuietHoursCalendarHtmlPrintableMultipage` helper restores per-page documents from the concatenated text; `detectEmptyRegions` flags trivial regions for skip.

### 3. i18n for the print warnings panel
Per-locale badge prefixes, severity labels, empty-state badge + label, "Printed" prefix, default footer text. Composes the base print render so the chip datum + monochrome layout stay consistent. The unused-destination chip has a structural quirk (base render echoes the English severity label in `cov-warn-label`); the i18n layer rewrites that span too for that severity only so a Spanish render doesn't leak English text. `detectCoverage` helper for the CI gate (10 expected keys). Caller-supplied `footerText` honoured verbatim. `extractLines` plain-text helper for log review.

### 4. Standalone audit-coverage report for anonymise results
Aggregates every QA signal the on-call reviews before mailing a third-party-safe manifest: name strategy, hash truncation, hash prefix, distinct patient + manifest + lookup counts, collision flag, redacted-row count + samples (capped at 5 by default), preserve-label flags, composite worst-wins verdict (`review-collisions > review-redacted > empty-cohort > ship-safe`). `detectLeakWarnings` returns PHI-leak warnings for preserve flags. `aggregate` rolls N reports (per ward / clinic) with sum'd counts, OR'd preserve flags (worst-wins for PHI leak), worst-wins verdict, `nameStrategy: 'mixed'` + null `hashHexLength` / `hashPrefix` when inputs disagree.

### 5. i18n for the search-input render
Per-locale placeholder, aria-label, and visually-hidden empty-state hint. Smallest i18n bundle in the package (3 keys). Composes the base anchored-search-input render so the data attributes + datalist + anchor map stay consistent. Graceful fallback to English. `detectCoverage` helper. `renderEmergencyCardSearchInputI18nMultiLocale` rolls the same TOC across N locale bundles for clinic-chain portals.
