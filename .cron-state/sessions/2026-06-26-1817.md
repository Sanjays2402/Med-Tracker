## Shipped (5)

1. **refills-runout-sort-persist** (f2f248c) — The Default / Soonest run-out
   sort on /refills now survives a reload via localStorage, matching the
   medications density + run-out-group prefs. New `lib/refill-sort-pref.ts`
   owns the storage key, default, and forward-compatible JSON
   parse/normalize/serialize guards (an unknown stored key falls back to
   default). 12 tests.

2. **refills-sort-soonest-headline** (a9a37f6) — When the runout sort is active,
   a tone-coded pill beside the control shows what goes dry next: "soonest
   overdue" / "next out today / tomorrow / in Nd", across the still-active
   refills (picked-up excluded). Added `formatSoonestRunout` +
   `soonestRunoutTone` to `lib/refill-sort` (reads the soonestDays the summary
   already computes). 8 new tests (21 in the suite).

3. **today-group-by-part-of-day-counts** (5108875) — Each
   Morning/Afternoon/Evening/Night header on /today carries a progress chip:
   "M of N taken" / "all N taken" (sage when done) / nothing when empty.
   Extracted the page's previously-inline hour bucketing into a tested
   `lib/part-of-day.ts` (`groupByPartOfDay` + `countDoses` + `sectionCountLabel`);
   buckets still follow the viewer's local time exactly as before. 17 tests.

4. **caregivers-expiry-pill** (2857a23) — Caregiver rows whose access expires
   within 7 days show an amber "Expires in Nd" (today/tomorrow/in Nd) pill with
   a relative-time tooltip, in place of the plain Active pill. Expired stays
   danger. New `lib/caregiver-expiry.ts` composes the existing isExpired /
   isExpiringSoon / relativeTime into one render-ready pill model (status + tone
   + label + daysUntil) with a configurable soon window. 16 tests.

5. **dashboard-trend-sparkline-real** (c069f64) — THE BIG ONE. The dashboard's
   two-week strip used hash-seeded pseudo-random wobble around the average —
   fabricated per-day variance the app never actually had (a direct SOUL.md
   "no fake stats" violation sitting in the UI). Replaced with an honest model:
   older cells carry the prior-window average, newer cells the current-window
   average, so the single step mirrors the real this-vs-prior trend arrow.
   No baseline -> flat strip + a caption that says it's the window average, not
   per-day data. New `lib/trend-series.ts`; `seriesFromDaily` is ready for a
   real per-day API series when one lands. 13 tests.
