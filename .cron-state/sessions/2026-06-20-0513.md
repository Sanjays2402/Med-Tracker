## Process notes

1. **Name conflict caught by LSP**: both `bp-log.ts` and `glucose-log.ts` initially exported `SummaryOptions`. The first attempt to wire them into `index.ts` failed the in-editor LSP check with `TS2308: has already exported a member`. I renamed both to `BpSummaryOptions` and `GlucoseSummaryOptions` BEFORE the first commit landed, so the broken-export state was never in git history. Pattern: any future per-module summary type should follow `<Module>SummaryOptions`.

2. **Test date subtlety**: `new Date('2026-06-15')` parses as UTC midnight, which on PDT (UTC-7) lands on June 14 23:00 local. Since the dose-history aggregator buckets in local time, all my "from / to / due" dates were off by one day, causing 6 tests to fail. Fix: use the 3-arg constructor `new Date(2026, 5, 15)` (month is 0-indexed) for local-time intent. Documented in the test file's helper comment.

3. **Critical-reminder batching bug**: my first implementation walked the sorted list and flushed the bucket whenever a critical reminder appeared. That correctly isolated the critical but ALSO orphaned the surrounding normals into separate batches. Fix: partition criticals from normals up front, batch the normals as a single sweep, emit each critical as a singleton, then merge-sort by `fireAt`. Tested.

4. **Per-commit slice rule**: each commit added exactly one source file, one test file, and the matching `export * from` line to `index.ts`. Pushed only ONCE at the end of the batch with a single push to `feature/autoship`.
