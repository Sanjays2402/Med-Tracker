## Design arc

The five features were chosen to form a coherent stack rather than
five orthogonal utilities:

1. **prescriber-directory** answers "who manages each of my meds?" by
   deduping prescribers across manual / EHR / pharmacy sources with
   NPI-Luhn + canonical-name keys.
2. **drug-class-coverage** answers "is my regimen missing a class my
   condition needs?" using curated bundles (CAD, HFrEF, DM2, COPD).
3. **prescription-fill-history** answers "did I have supply every day
   in the period?" using the cumulative "extend, don't reset" rule
   that naturally caps coverage at 1.0 per day.
4. **pdc-by-medication** composes #3 into the FDA-style PDC metric
   that PBMs and CMS Star Ratings use.
5. **pharmacy-fill-reconciliation** answers "did the pharmacy dispense
   what they were supposed to?" — short fills, late refills, early
   refills, dispensing errors.

Together these are the data layer for a "Refill & Adherence" tab in
the caregiver dashboard.

## Bugs caught

- prescriber-directory: `stripDecorations` was eating the structural
  comma in "Smith, Jane" because honorific stripping ran first.
  Reordered to detect the comma in the raw string before decoration
  strip; "Jane Smith, MD" still falls through to the space-form path
  because the right-of-comma side is empty after MD is stripped.
- prescription-fill-history: `intervals.length === 0` was the wrong
  sentinel for "first iteration" — the second fill always saw
  intervals as empty too (push-on-boundary). Fixed with explicit
  `initialized` flag.
- prescription-fill-history: default windowEnd of max-fillEnd across
  the regimen gave each medication a phantom trailing gap from
  unrelated longer-tail medications. Switched to per-medication
  default windows; shared windows still available when caller passes
  both bounds explicitly. PDC uses the explicit-window path.
- pharmacy-fill-reconciliation: initial classification order put
  late-refill before short-fill, so the test for a partial fill that
  arrived exactly when supply ran out misclassified. Reordered:
  duplicate -> quantity mismatches -> timing flags. Right design
  choice: a partial fill that LATER causes a late refill is still
  primarily a partial-fill problem.
- index.ts: `FillEvent` collision between refill-cost-projector (tick
  7) and prescription-fill-history (tick 8). Renamed the new one to
  `PharmacyFillEvent` — third instance of the `<Module>Foo` rename
  pattern established in ticks 4 + 6. Future collisions: prefix the
  NEW module's domain noun rather than touching the older type.
