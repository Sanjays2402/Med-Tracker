## Lessons captured for future ticks

1. **Fourth `<Module>Foo` rename in the autoship history.** This tick:
   `PdcTrendDirection` for the pdc-trend module to avoid colliding
   with weight-trend's pre-existing `TrendDirection` ('gaining' /
   'stable' / 'losing' / 'insufficient'). The `<Module>Foo` precedent
   is now load-bearing — any new module that wants a generic-sounding
   type name (TimeBucket, DoseHistoryEntry, FillEvent, TrendDirection)
   should prefix with its domain noun up front. Saves the post-tick
   fixup pass.
2. **TS strict + Array.sort() destructure.** `const [a, b] = [x, y].sort()`
   does not narrow to `[string, string]` under strict null checks:
   `Array<T>.sort()` returns `Array<T>` (length isn't carried). Cure:
   `const sorted = [x, y].sort(); const a = sorted[0]!; const b = sorted[1]!;`
   Two of my modules tripped this in tick 9; documenting it here so
   the next tick uses the explicit-index pattern from the start.
3. **PDC trend math: latest = shortest window, baseline = longest.**
   Counterintuitive for slope: a declining trend gives a POSITIVE
   slope (PDC rises with windowDays because older windows were
   healthier). The `direction` field is the authoritative human label;
   `slopePerDay` is the diagnostic number.
4. **inventory-low-stock-forecast composes two old modules without
   duplicating math.** Reuses `summarizeLots()` from inventory-ledger
   AND `dailyUsageFromSchedules()` from refill-forecast. Future
   composition-style features should follow this pattern: import the
   primitives, don't reimplement them.
5. **drug-class-coverage-bundles-builder is the first @med/utils
   module with ICD-10 routing.** Prefix-match against lower-cased
   ICD-10 strings is the right shape (codes are hierarchical:
   I25.10 starts with i25). The CONDITIONS table is intentionally
   short (10 conditions) and curated; clinical guideline drift means
   each addition needs a cited rationale string.
